=== KEY FINDINGS ===

1. LOGPROBS WORK. Real token probability distributions from Ollama produce
   meaningful escalation signals. tokenProbabilitySpread now ranges 0.02-0.55
   vs the Phase 1 uniform 0.30-0.50 from speed estimation.

2. VALENCE PRE-GATE PREVENTS WASTED WORK. Hard queries with high stakes
   skip directly to Tier 2 or Tier 3, avoiding a slow Tier 1 generation
   that would be thrown away. This saved 20-40s per hard query.

3. CONFIDENCE THRESHOLD TUNING IS CRITICAL. The Phase 1 threshold (0.65)
   was calibrated for the old fake signals. Phase 2 required recalibration
   to 0.28 (meaning escalation fires when confidence < 0.72). This is the
   most sensitive parameter in the system.

4. TIER 3 LOCAL IS VIABLE BUT SLOW. qwen2.5:32b on CPU takes 200-350s
   for a hard query. This is the expected behavior for NVMe-streamed
   inference without GPU. With GPU staging buffer (VRAM), this would be
   10-20x faster.

5. MEDIUM QUERY BORDERLINE ZONE. Queries with a single complexity keyword
   (e.g., "Explain X") produce valence 0.10-0.14 and confidence 0.76-0.83.
   Some escalate (spread > 0.48), some don't. This is the zone where
   the sleep analyst can learn to fine-tune thresholds.
