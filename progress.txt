WHAT WAS BUILT:
  1. Real logprob-based escalation signals (src/tiers/signals.ts)
     - tokenProbabilitySpread from entropy of top-k logprob distributions
     - semanticVelocity from sliding-window logprob variance
     - surpriseScore from low-probability token rate
     - attentionAnomaly from skip-bigram + trigram repetition
  2. Shared signal computation (src/tiers/signals.ts) — extracted from 3 tiers
  3. Valence-based pre-escalation gate (src/router/router.ts)
     - composite > 0.50 with stakes → Tier 3
     - composite > 0.35 → Tier 2
     - Prevents wasted Tier 1 generation on obviously complex queries
  4. Updated valence scorer (src/router/valence.ts)
     - High-complexity markers, clause counting, multi-part detection
     - Better separation: simple 0.00-0.02, medium 0.10-0.28, hard 0.42-0.55
  5. Tier 3 Local (src/tiers/tier3-local.ts)
     - NVMe-streamed via Ollama mmap, default qwen2.5:32b
     - GPU staging: transparent via Ollama, graceful degradation
     - Configurable keep_alive for layer caching
  6. Tier 3 Router (src/tiers/tier3.ts)
     - Prefers local model, falls back to Anthropic API
  7. Cold concept prewarmer (src/sleep/prewarmer.ts)
     - Runs after sleep analysis, probes concepts to warm model state
  8. Benchmark tier tracking (bench.ts, repl-test.ts)
     - Results now show actual tier, escalation path, confidence
  9. Bug fixes:

     - Async escalation logging (non-blocking in hot path)
     - REPL shows tier label in output

WHAT THE SELF-REVIEW FOUND THAT WAS UNEXPECTED:
  1. The semantic velocity signal was INVERTED in Phase 1 — high vocabulary
     diversity (the proxy used) indicated the model was doing WELL, not struggling.
     With real logprobs, this is replaced by logprob variance which correctly
     measures uncertainty fluctuation.
  2. The surpriseScore was always ~0 because it compared token count to
     eval_count, which are the same value. Completely non-functional.
  3. The router was not speculative at all — it generated the full response,
     then checked signals post-hoc with identical signals for every buffer window.
  4. The confidence threshold math was backwards — the Phase 1 threshold of 0.65
     meant escalation only fired when confidence < 0.35, which with the uniform
     0.58-0.83 signals was impossible.
